Defference between local storage and session storage.

No. Local Storage Session Storage
1 The storage capacity of local storage is 5MB/10MB The storage capacity of session storage is 5MB
2 As it is not session-based, it must be deleted via javascript or manually It’s session-based and works per window or tab. This means that data is stored only for the duration of a session, i.e., until the browser (or tab) is closed
3 The client can only read local storage The client can only read local storage
4 There is no transfer of data to the server There is no transfer of data to the server

Defference between global scope and block scope.

No. Global Scope Block Scope
1 Variables declared Globally (outside any function) have Global Scope. ES6 introduced two important new JavaScript keywords: let and const.
2 Global variables can be accessed from anywhere in a JavaScript program. These two keywords provide Block Scope in JavaScript.
3 Variables declared with var, let and const are quite similar when declared outside a block. Variables declared inside a { } block cannot be accessed from outside the block.

How does JS even loop work?

The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.
